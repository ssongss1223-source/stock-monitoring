## 에이전트별 상세 역할

### 1. Orchestrator
- **역할**: 전체 파이프라인 제어, 에이전트 간 데이터 전달
- **구현**: Python 클래스 + asyncio
- **GCP**: Cloud Scheduler → Flask/FastAPI 엔드포인트 or 직접 `python main.py` 실행 (e2-micro cron)

### 2. MarketFilterAgent
- **역할**: KOSPI/KOSDAQ 기술 스코어로 장세 판별 → `market_bias` 산출
- **구현**: 순수 Python + pykrx + pandas (Phase 1)
- **Phase 2**: 매크로 센티멘트(거시 뉴스, 금리/환율) + Claude Haiku 추가

**[market_bias — 매수 차단 아님, 등급 임계값 보정치]**
| 총점 | 장세 | market_bias | 의미 |
|---|---|---|---|
| 8점↑ | 상승장 ✅ | 0 | 등급 기준 그대로 |
| 5~7점 | 횡보장 ⚠️ | +2 | 같은 등급 받으려면 +2점 더 필요 |
| 4점↓ | 하락장 ❌ | +5 | 매우 강한 신호만 통과 (차단 아님) |

- **출력**: `{ market: "bull"|"sideways"|"bear", score: int, market_bias: int }`

### 2-B. UniverseManager
- **역할**: 분석 대상 종목 목록 결정
- **모드**: `"watchlist"` / `"top50_mktcap"` / `"sector_top5"` (기본값)
- `INCLUDE_WATCHLIST=True`이면 모드와 관계없이 관심종목 항상 포함
- **GCP**: `data/portfolio.json`, `data/watchlist.json` → GCS 버킷 또는 VM 로컬 파일

### 3. TechnicalAnalysisAgent
- 다중 이동평균 스코어 (정배열), 일목균형표, RSI/MACD/BB
- 패턴 인식: 컵앤핸들, 밥그릇, 삼각수렴, BB수축
- 지지/저항 겹침 분석 (피보나치, 매물대 POC)
- **출력**: `{ trend_score: int, pattern: str|None, support: float, resistance: float }`

### 4. VolumeAnalysisAgent
- 거래량 폭발 예측 스코어 (7개 조건), OBV, 외국인/기관 수급, 공매도 잔량
- **출력**: `{ volume_score: int, explosion_imminent: bool, smart_money_flow: str }`

### 5. BuySignalAgent
- 앙상블 합산 → S/A/B 등급 산출 (market_bias 페널티 적용)
- 참고 손절가(-5% or 구름대 이탈), 참고 목표가(매물대 저항), 손익비 계산
- **등급 기준**: S급 추세12↑+거래량12↑+패턴, A급 추세8↑+거래량8↑, B급 형성 중

### 6. SentimentAgent *(Phase 2)*
- 네이버/다음 뉴스 RSS → Claude Haiku 감성 분류
- DART API 공시 감지, YouTube Data API 섹터 관심도

### 7. SellSignalAgent
- 익절/추세이탈/강제손절 시나리오 스코어링 (전략서 9장)
- 강제손절(-5% or 60일선 이탈)은 점수 무관 즉시 발동
- **항상 실행** (장세 무관)

### 8. ReportAgent
- S/A/B 등급순 매수추천 + 매도신호 우선순위순 종합
- 텔레그램 봇 API로 발송 (전략서 12장 포맷)

## 유연한 스코어링 시스템 (핵심 설계)

### 원칙
모든 점수 기준을 YAML로 외부화 → 코드 수정 없이 파라미터 변경 가능.
백테스트 후 `v2_conservative/` 폴더만 만들어 A/B 비교.

```
config/scoring/
├── v1_baseline/       # 전략서 초기값
│   ├── market.yaml
│   ├── volume.yaml
│   ├── technical.yaml
│   ├── buy_grade.yaml
│   └── sell.yaml
└── v2_*/              # 백테스트 후 조정 버전
```

```yaml

rules:
  - id: vol_5d_above_20d
    points: 2
    description: "5일 거래량 평균 > 20일 평균"
  - id: vol_consecutive_3d
    points: 3
    description: "3일 이상 연속 거래량 증가"
  - id: vol_rise_price_flat
    points: 4
    description: "주가 횡보 중 거래량 증가 (세력 매집)"
  - id: obv_uptrend_price_flat
    points: 3

  - id: vol_vs_52w_low_2x
    points: 2

  - id: foreign_inst_buy
```

### ScoringEngine 구조
```python
class ScoringEngine:
    def __init__(self, config_dir: str = "config/scoring/v1_baseline/"):
        self.rules = self._load_all_yaml(config_dir)

    def score_volume(self, data) -> VolumeResult: ...
    def score_technical(self, data) -> TechnicalResult: ...
    def determine_grade(self, tech, vol, market_bias: int) -> str: ...
```

각 에이전트는 ScoringEngine을 주입받아 사용 — 에이전트 코드 자체는 변경 없음.

```python

class MarketContext:
    market_status: str    # "bull" | "sideways" | "bear"
    score: int
    market_bias: int      # bull=0, sideways=+2, bear=+5

@dataclass
class BuySignal:
    ticker: str
    name: str
    grade: str            # "S" | "A" | "B"

    trend_score: int
    volume_score: int

    stop_loss: float
    target_price: float

class SellSignal:
    ticker: str
    action: str           # "stop_loss" | "full_sell" | "half_sell" | "hold"
    priority: int         # 전략서 9-5 우선순위 1~5
    score: int
    reason: str
```

## Orchestrator 실행 흐름

```python
async def run_daily(scoring_config: str = "config/scoring/v1_baseline/"):
    engine = ScoringEngine(scoring_config)

    # 장세 판단 + 매도신호 병렬 시작
    market_task = asyncio.create_task(MarketFilterAgent(engine).run())
    sell_task = asyncio.create_task(SellSignalAgent(engine).run(portfolio))

    market = await market_task

    # 유니버스 결정 후 종목별 병렬 분석
    candidates = UniverseManager().get_universe()
    results = await asyncio.gather(*[analyze_stock(t, market, engine) for t in candidates])

    # B급 이상만 (하락장 market_bias=+5 → 자연스럽게 추천 수 감소)
    buy_signals = [r for r in results if r.grade in ("S", "A", "B")]

    sell_signals = await sell_task
    await ReportAgent().send(market, buy_signals, sell_signals)
```

## GCP 배포 설계

### 권장 구성: e2-micro VM (무료 티어)

```
GCP e2-micro (us-central1, 무료)
├── Python 3.11 + venv
├── main.py + APScheduler (매일 16:00 KST = 07:00 UTC)
├── Secret Manager → 환경변수 주입 (텔레그램 토큰 등)
└── data/ 폴더 → VM 로컬 저장 (portfolio.json, watchlist.json)
```

**무료 티어 범위**: e2-micro 월 730시간 (1개 인스턴스 상시 무료), 아웃바운드 트래픽 1GB/월

### 환경변수 관리 (GCP Secret Manager)
```bash
# 민감 정보는 Secret Manager에 저장
gcloud secrets create TELEGRAM_BOT_TOKEN --data-file=./token.txt
gcloud secrets create TELEGRAM_CHAT_ID --data-file=./chat_id.txt
# VM 시작 시 자동 주입 (startup script)
```

### 배포 파일 구조
```
deploy/
├── startup.sh          # VM 초기 설치 스크립트
├── update.sh           # 코드 업데이트 + 재시작
└── Dockerfile          # 향후 Cloud Run 이전 대비
```

### 로컬 → GCP 동기화
- **개발**: 로컬 Windows (Miniforge + VS Code)
- **배포**: `git push` → VM에서 `git pull` + `systemctl restart stock-monitor`
- `systemd` 서비스로 재부팅 시 자동 시작

### 향후 Cloud Run 이전 (선택적 업그레이드)
e2-micro 리소스 부족 시:
- `Dockerfile` 빌드 → Artifact Registry 푸시
- Cloud Scheduler → Cloud Run Job 트리거 (실행 중에만 과금)

## 검증 계획

1. **단위**: 각 에이전트 개별 실행 → 스코어 출력 확인
2. **장세 판단**: 2025년 4월 폭락기 데이터 → `market_bias=+5` + 추천 수 급감 확인
3. **백테스트 버전 비교**: v1_baseline vs v2_* YAML로 동일 기간 결과 비교
4. **백테스트**: vectorbt 2025-04 ~ 2026-04 → 오신호 비율, 평균 손익비
5. **텔레그램**: 실제 봇으로 매수추천 + 매도신호 포맷 확인
