# 거래량 기반 종목별 행동 프로파일링 시스템 설계서

---

# 1. 목적

본 시스템의 목적은 60분봉 데이터를 기반으로 종목별 거래량·가격·수급 행동 패턴을 분석하여,  
**향후 3거래일 이내 3% 이상 상승 가능성이 높은 종목 후보를 매일 08시에 제공**하는 것이다.

핵심 방향은 단순 거래량 증가 탐지가 아니라,  
종목별 평소 행동 대비 비정상적인 수급 이벤트를 포착하는 것이다.

---

# 2. 시스템 개념

```text
과거 60분봉 데이터 수집
→ 종목별 행동 프로파일 생성
→ 전일 이상 수급 이벤트 탐지
→ 과거 유사 패턴 성공률 계산
→ 3일 내 3% 상승 가능성 점수화
→ 매일 08시 후보 종목 출력
```

---

# 3. 핵심 가설

```text
1. 종목마다 고유한 거래량·가격 행동 패턴이 존재한다.

2. 거래량은 절대값보다 종목별 평소 거래량 대비 상대적 위치가 중요하다.

3. 60분봉 기준 거래량 변화는 단기 수급 유입을 포착하는 데 활용 가능하다.

4. 이상 거래량이 가격 돌파, OBV 상승, VWAP 상회와 동시에 발생하면
   단기 상승 확률이 높아질 수 있다.

5. 목표는 3거래일 이내 3% 이상 상승할 후보 종목을 찾는 것이다.
```

```text

```

```text
전일 수급 이상 기반 익일 관심종목 선별 시스템
```

으로 정의한다.

# 5. 주요 Feature 설계

# 5.1 거래량 Feature

## 1) same_time_volume_ratio

### 정의

```text
현재 60분봉 거래량 / 최근 N일 동일 시간대 평균 거래량
```

### 목적

```text
해당 종목의 평소 시간대별 거래량 대비 이상 거래량 탐지
```

---

## 2) volume_zscore

### 정의

```text
volume_zscore =
(현재 거래량 - 평균 거래량) / 표준편차
```

### 목적

```text
현재 거래량이 평균 대비 몇 표준편차 높은지 측정
```

---

## 3) relative_turnover

### 정의

```text
거래대금 / 시가총액
```

또는:

```text
거래량 / 유통주식수
```

### 목적

```text
종목 규모 대비 실제 수급 강도 측정
```

---

## 4) volume_acceleration

### 정의

```text
현재 60분봉 거래량 / 직전 60분봉 거래량
```

### 목적

```text
거래량 증가 속도 탐지
```

---

# 5.2 가격 Feature

## 5) intraday_breakout_strength

### 정의

```text
(종가 - 단기 저항선) / ATR
```

### 목적

```text
저항선을 얼마나 강하게 돌파했는지 측정
```

---

## 6) box_breakout_distance

### 정의

```text
(종가 - 박스권 상단) / 박스권 높이
```

### 목적

```text
박스권 상단 근접 또는 돌파 여부 측정
```

---

## 7) VWAP distance

### 정의

```text
(종가 - VWAP) / VWAP
```

### 목적

```text
평균 매입 단가 대비 현재 가격 위치 측정
```

---

## 8) MA alignment

### 정의

```text
MA5 > MA20 > MA60 여부
```

### 점수화 예시

```text
MA5 > MA20: +1
MA20 > MA60: +1
종가 > MA20: +1
```

---

# 5.3 수급 흐름 Feature

## 9) OBV slope

### 정의

```text
최근 N개 60분봉 OBV의 회귀 기울기
```

### 목적

```text
가격보다 먼저 수급이 들어오는지 확인
```

---

## 10) 체결강도

### 정의

```text
매수체결량 / 매도체결량
```

---

## 11) 매수/매도 imbalance

### 정의

```text
(매수잔량 - 매도잔량) / 전체잔량
```

---

## 12) 호가 변화 속도

### 정의

```text
단위 시간당 호가 변경 횟수
```

---

# 6. 종목별 Threshold 산출 방식

각 Feature는 고정 기준이 아니라 종목별 분포 기반으로 판단한다.

---

## 6.1 Percentile 방식

### 예시

```text
same_time_volume_ratio가
해당 종목의 최근 6개월 분포에서
상위 95% 이상이면 이상 신호
```

### 기준

```text
P90 = 관심
P95 = 이상 신호
P99 = 강한 이상 신호
```

---

## 6.2 Z-score 방식

### 예시

```text
volume_zscore > 2.0 : 이상 거래량
volume_zscore > 3.0 : 강한 이상 거래량
```

---

## 6.3 Rolling 기간

### 추천 기준

```text
최근 20일: 단기 상태
최근 60일: 중기 기준
최근 120일: 종목 기본 리듬
```

최종 판단은 60일 또는 120일 분포를 기본으로 하고,  
최근 20일 변화도 함께 본다.

# 7. 이벤트 탐지 로직

전일 60분봉 중 아래 조건을 만족하는 봉을 탐지한다.

## MVP 조건 예시

```text
same_time_volume_ratio >= 해당 종목 P95
volume_zscore >= 2.0
종가 > VWAP
종가 > MA20
OBV slope > P70
```

### 추가 조건

```text
전고점 근처
박스권 상단 근처
60분봉 양봉
윗꼬리 과도하지 않음
```

---

# 8. 라벨 정의

목표가 3일 이내 3% 상승이므로 라벨을 아래처럼 만든다.

## 라벨 정의

```text
signal_date = 이상 신호 발생일
entry_price = 다음 거래일 시가

target =
이후 3거래일 내 고가가
entry_price 대비 +3% 이상 도달하면 1
아니면 0
```

### 예시

```text
2026-05-01 이상 신호 발생
2026-05-02 시가 진입 가정

2026-05-02 ~ 2026-05-06 중
고가가 +3% 이상 도달

→ target = 1
```

# 9. 점수화 방식

MVP에서는 ML보다 Rule-based Score를 먼저 추천한다.

## 예시

```text
volume_score =
same_time_volume_ratio percentile 점수

breakout_score =
박스권/저항 돌파 점수

vwap_score =
VWAP 상회 점수

obv_score =
OBV slope 점수

ma_score =
이동평균 정렬 점수
```

---

## 최종 점수

```text
total_score =
0.35 * volume_score
+ 0.25 * breakout_score
+ 0.20 * obv_score
+ 0.10 * vwap_score
+ 0.10 * ma_score
```

# 11. 개발 단계

# Phase 1: 데이터 수집

```text
60분봉 OHLCV 수집
종목별 저장
일별 업데이트
결측치 검증
액면분할/무상증자 보정
```

---

# Phase 2: Feature 생성

```text
거래량 feature 생성
가격 feature 생성
OBV/VWAP/MA 계산
종목별 percentile 계산
```

---

# Phase 3: 이상 이벤트 탐지

```text
전일 60분봉 분석
종목별 threshold 비교
이상 이벤트 저장
```

---

# Phase 4: 백테스트

```text
이상 이벤트 발생 후
다음 거래일 시가 기준 진입
3거래일 내 +3% 도달 여부 검증
최대 낙폭 확인
손절 기준 검토
```

---

# Phase 5: 08시 리포트

```text
후보 종목 랭킹
유사 패턴 성공률
주요 feature 표시
매수 후보/관찰 후보 구분
```

# 13. 주의사항

가장 조심해야 할 것은 과최적화다.

```text
특정 기간에는 잘 맞지만
다른 기간에는 성과가 무너질 수 있음
```

반드시 아래를 검증해야 한다.

```text
상승장/하락장/횡보장 분리 검증
대형주/중형주/소형주 분리 검증
거래대금 하위 종목 제외
뉴스/테마 급등주 예외 처리
손절 기준 포함
```

# 14. 결론

이 시스템은 단순히 거래량 많은 종목을 찾는 시스템이 아니다.

정확한 정의는 다음과 같다.

```text
종목별 평소 행동 패턴 대비
비정상적인 거래량·가격·수급 이벤트를 탐지하고,

그 이벤트가 과거에
3거래일 이내 3% 상승으로 이어졌는지를 기반으로

매일 08시 관심종목을 선별하는 시스템
```

---

## MVP 핵심

```text
전일 60분봉 이상 수급 이벤트
+ 종목별 percentile threshold
+ 다음 3거래일 +3% 백테스트
+ 매일 08시 후보 종목 랭킹
```
