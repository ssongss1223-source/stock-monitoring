# 작업 컨텍스트 불러오기

`docs/checkpoint.md`가 있으면 읽고, 현재 프로젝트 상태를 간단하고 실무적으로 요약한다.

## 목적
이 명령은 **컨텍스트 파악 전용**이다.

- 구현 시작 금지
- 파일 수정 금지
- 실제 작업 수행 금지
- 상태 파악과 다음 선택지를 정리하는 데만 집중

내가 빠르게 현재 상태를 이해하고, 다음 행동을 직접 결정할 수 있도록 돕는 것이 목적이다.

## 할 일
1. `docs/checkpoint.md`를 읽는다.
2. 현재 가장 중요한 목표를 파악한다.
3. 현재 진행 상태를 짧고 명확하게 요약한다.
4. 최근 완료한 핵심 작업을 정리한다.
5. 아직 남아 있는 핵심 작업을 정리한다.
6. 막힌 점, 가정, 리스크를 정리한다.
7. 내가 선택할 수 있도록 다음 액션 후보를 제안한다.
8. 구현이나 수정을 시작하지 않는다.

## 규칙
- `docs/checkpoint.md`가 없으면 없다고 명확히 말한다.
- **Last Updated가 오늘 기준 3일 이상 지났으면**, 요약 상단에 `> ⚠ checkpoint가 N일 경과됨 — VM/DB 상태를 직접 확인할 것` 경고를 출력한다.
- checkpoint 내용이 오래됐거나, 불완전하거나, 현재 저장소 상태와 안 맞아 보이면 그 점을 명확히 지적한다.
- checkpoint나 저장소에서 확인되지 않은 전제들을 추측해서 쓰지 않는다.
- 요약은 짧고 실용적으로 유지한다.
- 어떤 구현도 시작하지 않는다.
